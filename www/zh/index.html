工作原理

钱包 + 网关 = 微支付堆栈

AI代理访问门控API，收到402发票，通过ClawPurse链上支付，并带证明重试。网关在Neutaro上验证交易并代理到上游。无需账户，无需API密钥——仅链上结算。

  AI代理 / 脚本 / 浏览器
    │
    ├── GET /api/data ───────────────────▶  402网关
    │                                            │
    │   ◀── 402 Payment Required ──────────────┘
    │       { invoiceId, amount, address, memo }
    │
    ├── clawpurse send   --memo 
    │       （通过钱包进行链上NTMPI支付）
    │
    ├── GET /api/data ───────────────────▶  402网关
    │   Header: X-Payment-Proof:    │
    │                                            │
    │   网关在Neutaro链上验证tx  │
    │                                            │
    │   ◀── 200 OK（从上游代理） ──┘
钱包

ClawPurse钱包功能

本地管理NTMPI所需的一切——为自主代理内置护栏。

加密密钥库

使用scrypt密钥派生的AES-256-GCM。助记词锁定在本地节点上。

护栏向导

交互式设置发送限额、确认阈值和白名单强制模式。

目标白名单

使用每目标上限、必填备注和注释强制或记录可信地址。

交易收据

每笔发送本地记录tx哈希、区块高度、备注和护栏状态，提供完整的审计追踪。

质押与委托

委托、解除委托、重新委托NTMPI。发现验证者、追踪奖励并管理22天的解绑期。

编程API

将钱包函数直接导入代理和脚本。send()、getBalance()、delegate()等。

402网关

用NTMPI微支付变现任何API

ClawPurse 402网关位于您的上游API前端，处理发票、链上验证和代理。两种支付流程、路由级定价和JWT保护的管理API。

按请求付费

客户端收到402发票，链上支付，带X-Payment-Proof头重试。网关验证并代理。

预付余额

客户端一次性存入NTMPI。后续请求使用X-Client-Id，网关按调用扣除——无需重复链上交易。

链上验证

查询Neutaro REST API确认支付金额、备注和区块确认。

路由级定价

默认每请求价格加路由级覆盖。便宜的读取，昂贵的写入。

Docker和裸机部署

单命令docker compose up或Ubuntu部署脚本。SQLite存储，零外部依赖。

完整的API参考、配置指南和代理集成示例请访问 gateway.clawpurse.ai

生态系统

由Timpi和Neutaro区块链驱动

ClawPurse是Timpi独立搜索网络的支付层。Timpi结合了隐私优先搜索、开放数据API和奖励TAP网络运营商的DePIN节点。

Timpi技术栈

搜索、数据、DePIN

  • 搜索引擎：无用户画像或追踪的抗审查结果。
  • 数据API：对去中心化索引的编程访问。
  • DePIN节点：赚取NTMPI的Collectors、Guardians和GeoCores。

加入 timpi.io.

如何获取NTMPI

MEXC现货交易对

  1. 登录MEXC并存入USDT。
  2. 打开NTMPI/USDT现货市场。
  3. 下单，然后提取到您的Neutaro地址。

Gravity Bridge → Osmosis

  1. 通过Gravity Pulse从以太坊桥接稳定币。
  2. 通过Keplr或Leap进行IBC跳转到Osmosis。
  3. 兑换为NTMPI并IBC发送到Neutaro。
